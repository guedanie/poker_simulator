Shiny for Python: Express vs Core

Two ways to write the same app — pick the style that feels right

from shiny.express import input, render, ui
from simulation import run_sim

ui.input_action_button("run", "Run Simulation")
@render.text

return run_sim(input.cards())

Express mode (simpler)

Write UI and server logic together, top to bottom. Less boilerplate, easier to get started. Newer syntax — introduced in 2023.

Pros

  • Less code
  • Easier to read for beginners

Cons

  • Harder to scale to large apps
  • Less like R Shiny structure

Core mode (recommended)

Explicit separation of UI and server — the same mental model as R Shiny. More verbose but cleaner structure for an app this size.

Pros

  • Mirrors R Shiny exactly
  • Scales cleanly
  • Standard Posit Connect pattern

Cons

  • More boilerplate upfront
